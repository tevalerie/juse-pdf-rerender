J-USE PDF Re-Render Tool

Internal · Read-only
What this is: a stand-alone tool to regenerate the camera-ready J-USE REOI 2026 PDF from raw form data — useful for re-rendering historical submissions after a renderer fix, or for archival. It does not call any backend, does not increment the production counter, does not touch the Sheet, and does not send emails. Everything runs locally in your browser; you save the PDF wherever you want it.

1. Provide the application data

Three ways to load an application:

A. Fetch by Reference Number (recommended)

Enter the application's JUSE-2026-XXXXXX reference. Pulls the complete enriched data from the Sheet + Drive (master scalars, outcome metrics, benefit traceability, documents with file metadata, enablers, TNA scores, implementation roles).

Read-only call to the Apps Script rebuild endpoint. Does not touch the production submission counter, doPost URL, Sheet writes, or emails.

⏱ Original "Generated on …" time (optional override)

Only needed for historical submissions (JUSE-2026-000005 … 000008). Read the date + time off the original PDF's footer and enter it as Jamaica local time. Leave blank for any submission rendered after Task C went live — the form bakes in its own timestamp.
